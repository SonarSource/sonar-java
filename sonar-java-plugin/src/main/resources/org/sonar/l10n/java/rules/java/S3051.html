Why is this an issue?

A method named main with an invalid signature for a program entry point is confusing. It suggests the method is intended to be an entry point, but the JVM will not recognize it as such.

Valid signatures for a program entry point are:

  • public static void main(String[] args) - traditional signature
  • static void main(String[] args) - static main without public modifier (Java 21+)
  • void main(String[] args) - instance main method (Java 21+)
  • void main() - main method without parameters (Java 21+)
  • static void main() - static main without parameters (Java 21+)

Noncompliant code example

class App {
  // Wrong return type
  int main(String[] args) { // Noncompliant
    return 0;
  }

  // Wrong parameter type
  void main(int[] args) { // Noncompliant
  }

  // Too many parameters
  void main(String[] args, int extra) { // Noncompliant
  }
}

Compliant solution

class App {
  public static void main(String[] args) { // Compliant
  }

  // Or using Java 21+ flexible main methods:
  void main() { // Compliant
  }
}

Exceptions

Methods named main in interfaces, abstract classes, and annotations are ignored, as these cannot be used as program entry points.
