This rule is deprecated, and will eventually be removed.

Why is this an issue?

Failing to explicitly declare the visibility of a member variable could result it in having a visibility you donâ€™t expect, and potentially leave it open to unexpected modification by other classes.

Noncompliant code example

class Ball {
    String color="red";  // Noncompliant
}
enum A {
  B;
  int a;
}

Compliant solution

class Ball {
    private String color="red";  // Compliant
}
enum A {
  B;
  private int a;
}

Exceptions

Members annotated with @VisibleForTesting annotation are ignored, as it indicates that visibility has been purposely relaxed to make the code testable.

class Cone {
  @VisibleForTesting
  Logger logger; // Compliant
}
