Shared naming conventions allow teams to collaborate efficiently. This rule raises an issue when a test class name does not match the provided regular expression.

Note that test class is only a top level class that contains a test annotated methods or nested annotated classes.

Noncompliant Code Example

With the default value: ^((Test|IT)[a-zA-Z0-9]+|[A-Z][a-zA-Z0-9]*(Test|IT|TestCase|ITCase))$

class Foo {  // Noncompliant
  @Test
  void check() {
  }
}

class Bar {  // Noncompliant
  @Nested
  class PositiveCase {
    @Test
    void check() {
    }
  }
}

Compliant Solution

class FooTest {
  @Test
  void check() {
  }
}

class BarIT {
  @Nested
  class PositiveCase {
    @Test
    void check() {
    }
  }
}
