The compiler automatically initializes class fields to their default values before setting them with any initialization values, so not only is there no need to explicitly set a field to its default value, but doing so is slightly less performant. Further, under the logic that cleaner code is better code, it's considered poor style to do so.

Noncompliant Code Example

public class MyClass {

  int count = 0;  // Noncompliant
  // ...

}

Compliant Solution

public class MyClass {

  int count;
  // ...

}
