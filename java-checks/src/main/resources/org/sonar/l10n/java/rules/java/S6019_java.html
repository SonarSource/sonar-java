When a reluctant quantifier (such as *? or +?) is followed by a pattern that can match the empty string or directly by the end of the regex, it will always match the empty string when used with methods that find partial matches (such as find, replaceAll, split etc.) and behaves indistinguishably from a greedy quantifier when used with methods that find full matches.

This is likely a sign that the regex does not work as intended.

Noncompliant Code Example

Pattern.compile(".*?x?"); // Noncompliant, this will behave like "x?" when finding substrings and like ".*" when finding whole matches

Compliant Solution

Pattern.compile(".*?x");
